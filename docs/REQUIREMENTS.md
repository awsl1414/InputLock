# InputLock 需求文档

## 产品定位

**InputLock** 是一个 macOS 原生输入法锁定工具，用于锁定当前输入法，防止输入法被系统自动切换。

这是一个输入法管理/控制工具，而非输入法本身。

## 产品形态

程序以**菜单栏应用**形态运行（无 Dock 图标，无主窗口）。

## 功能需求

### 1. 状态栏图标与菜单

- 在 macOS 菜单栏显示应用图标
- 图标应能反映当前锁定状态（如锁定/未锁定）
- 点击图标弹出功能菜单，菜单结构如下：

```
┌─────────────────────────────┐
│ 🔒 输入法锁定: 开启/关闭     │  ← 总开关，点击切换锁定功能
├─────────────────────────────┤
│ ▸ 选择锁定输入法              │  ← 子菜单，列出系统已启用的输入法
│   ● 简体中文拼音             │     当前选中项前显示勾选标记
│     美式英文                  │
│     ...                     │
├─────────────────────────────┤
│ ✓ 登录时自动启动              │  ← 切换项，控制是否加入登录项
├─────────────────────────────┤
│ 关于                        │  ← 弹出关于窗口
├─────────────────────────────┤
│ 退出                        │
└─────────────────────────────┘
```

### 2. 输入法选择

- 菜单中列出系统已启用的输入法（用户在系统设置中添加的）
- 用户选择要锁定的输入法后，该项前显示勾选标记
- 锁定开关开启时，选中项即为被锁定的输入法

### 3. 锁定行为

**锁定目标：对抗 macOS 系统的自动输入法切换行为**（如切换窗口/应用时，系统根据上下文自动切换输入法）。

**不干预用户主动切换**：用户通过快捷键（Caps Lock、Ctrl+Space 等）主动切换输入法时，应当正常响应，不做拦截。

**不影响输入法内部的中英文切换**。

具体效果：
- 锁定后，当系统自动切换输入法时（如窗口焦点变化触发的自动切换），程序检测到变化并立即切换回锁定的输入法
- 用户主动按快捷键切换输入法时，程序识别为用户意图，放行并更新锁定目标为新选择的输入法
- 输入法自身的中文/英文模式切换不受影响

### 4. 登录时自动启动

- 菜单中提供"登录时自动启动"选项，默认关闭
- 开启后，将应用注册为 macOS 登录项（使用 `SMAppService` API，macOS 13+）
- 状态持久化，重启后记住用户选择

### 5. 首次启动授权指引

首次启动时，应用需要引导用户授予必要的系统权限。通过弹出引导窗口展示：

1. **Accessibility（辅助功能）权限**
   - 用途：检测输入法切换来源
   - 引导：提示用户前往 系统设置 → 隐私与安全性 → 辅助功能，将 InputLock 添加到允许列表
   - 检测：通过 `AXIsProcessTrusted()` 实时检测授权状态

授权流程：
- 首次启动时弹出引导窗口，说明权限用途
- 提供"打开系统设置"按钮，直接跳转至对应设置页面
- 授权完成后自动关闭引导窗口，进入正常工作状态
- 若用户跳过授权，菜单栏图标显示未授权状态，功能不可用

### 6. 退出

- 菜单底部提供"退出"选项
- 退出后释放所有监听和事件拦截资源

### 7. 关于

- 菜单中提供"关于"选项，点击弹出关于窗口
- 关于窗口内容：
  - 应用图标
  - 应用名称：InputLock
  - 版本号（如 v1.0.0）
  - 版权信息
  - 项目主页链接（如有）
